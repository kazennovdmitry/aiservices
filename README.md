AIServices — Dual-Protocol AI Agent Infrastructure in .NET 9
An open-source .Net solution that demonstrates how to expose the same AI tooling through two different integration protocols simultaneously:   

Structure:                                                                                                                                                                                                                                                                           
Three projects, one shared core:                                                                                                                                                                                                                                                                      
• CoreLogic — Shared domain library containing FileSystemPlugin (disk space analysis). The single source of truth for file system operations, consumed by both protocol implementations.                                                                                                              
• SemanticKernelAgent — ASP.NET Core Web API (POST /api/agent/chat) that routes user prompts through an LLM with automatic tool calling. Uses Semantic Kernel 1.76 with OpenAI-compatible connectors, targeting local LLMs (Ollama, LM Studio) by default.                                            
• MCPServer — A Model Context Protocol server (stdio transport) exposing the same GetDiskSpaceInfo tool via MCP, compatible with Claude Desktop, Cursor, and any MCP-native AI assistant.                                                                                                             
                                                                                                                                                                                                                                                                                                           
Key highlights                                                                                                                                                                                                                                                                                     
• Zero duplication — CoreLogic.Plugins.FileSystemPlugin is used once and exposed through both Semantic Kernel's [KernelFunction] and MCP's [McpServerTool] attributes                                                                                                                                 
• Protocol agnosticism — the same domain logic ships over REST/HTTP and stdio/MCP without modification                                                                                                                                                                                                
• Local-first design — defaults to localhost:8080/v1 (I used llama.cpp and local models), no cloud dependency required                                                                                                                                                                                                
• Clean architecture — .NET 9, implicit usings, centralized package management via Directory.Packages.props, generic host pattern with DI                                                                                                                                                             
                                                                                                                                                                                                                                                                                                           
Tech stack                                                                                                                                                                                                                                                                                         
C# / .NET 9 · Semantic Kernel 1.76 · ModelContextProtocol 1.3 · OpenAI-compatible connectors · ASP.NET Core · Microsoft.Extensions.Hosting                                                                                                                                                            
Open source. Local-first. Protocol-agnostic. Built to prove that the same AI capability shouldn't need two implementations
